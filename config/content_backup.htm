這個專案主要目的在測試可不可能單一 CMSiMDE 倉儲的內容能夠完全寄存在 Github 與 Fossil SCM.

Fossil SCM 設定

為了要讓 Fossil SCM 倉儲在 fossil add . 執行時可以納入 .git 目錄與相關以 . 開頭命名的特殊設定檔案, 必須在Fossil SCM 網際 settings 中勾選 dotfiles (V) 選項.

另外, 為了讓 CMSiMDE 靜態網頁能夠在 Fossil SCM doc project documentation 中跨網站擷取 https://fonts.googleapis.com 上的 .js 檔案, 則必須在 Fossil SCM 網際 settings 中的 default-csp 欄位中加入 https://fonts.googleapis.com, 詳細說明請參考 https://fossil-scm.org/home/doc/trunk/www/defcsp.md

Github 設定

在 Fossil SCM 倉儲與 Github 系統進行整合的唯一隱憂就是 _FOSSIL_ 檔案存放 Fossil SCM 的各種版本資料, 若檔案大小超過 50 MB, 可能就無法存放到 Github, 或許必須付費才能解決問題?
